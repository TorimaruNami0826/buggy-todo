### 03_TodoアプリにCRUD操作を実装
設計メモに基づき、curl.exe を使用してローカル環境のAPIエンドポイント（POST /todos）の動作検証を実施。

失敗事例
1. PowerShellでのコマンド分割エラー
事象: 画像の設計メモ通りに \（バックスラッシュ）で改行してコマンドを打つと、-H や -d が別コマンドと認識されエラーになった。
原因: WindowsのPowerShellでは \ は改行コードとして機能せず、コマンドが途切れてしまうため。
解消: 全てのオプションを改行せず、1行に繋げて記述することで解消。

2. JSONデータのパースエラー (SyntaxError)
事象: SyntaxError: Unexpected token... や Expected property name が返り、サーバー側でパースに失敗した。
原因: Windowsの curl.exe では、JSON内のダブルクォーテーションを適切にエスケープ（\"）する必要がある。また、外側を ' ' で囲むと挙動が不安定になる場合がある。
解消: 外側を " "（または特定の条件下で ' '）で囲み、中のダブルクォーテーションを \" とエスケープして送るように修正。
    *OK例:* -d "{\"title\":\"ClaudeCode\"}"

3. サーバー側での500エラー
事象: {"error":"サーバーエラーが発生しました"} が返却された。
原因: リクエストはサーバーに届いたが、DB接続やバリデーション（title必須チェック）など、サーバー内部の処理で例外が発生した。
解消: サーバーログ（ターミナルの出力）を確認し、データベースの起動状態やテーブル定義、バリデーションロジックを見直して修正。

4. URL/ホスト名の誤認
事象: curl: (3) URL rejected: Bad hostname 等のエラー。
原因: コマンドの書き方のミス（不要なスペースやエスケープ漏れ）により、後ろの文字列がURLの一部だと誤認された。
解消: コマンドを1行に整形し、不要な記号を削除して実行。
