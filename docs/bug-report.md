バグレポート

バグ1
症状： TODOの完了チェックボックスを押下した際、チェックができない
原因： src/routes/todos.js の70行目で、画面から送られてきた完了状態の値を保存する際に、必ず値を反転させてしまう記述（!completed）になっていたため
修正内容：  !completed → completedに修正
なぜこのバグが起きたか： !completed という否定演算子が誤って書かれていたため

バグ2
症状： 空欄で追加ボタンを押下した際、エラーメッセージが表示されるときとされないときがあり、表示が持続されない
原因： src/index.js の11行目で express.json() の設定はあるものの、不正なリクエストが送られてきた際、バリデーションの仕組みがうまく動いていなかったため
修正内容： express.json() のパース失敗を捕捉するエラーハンドラーを追加。HTMLではなくJSON(400)で返すようにした
なぜこのバグが起きたか： エラー入力時の対処方法を想定していなかったため

バグ3
症状： 存在しないTODOのIDを直接指定してアクセスしたり削除しようとした際、エラーにならずに成功したような挙動になる
原因： src/routes/todos.js の86行目で、指定されたIDのレコードが本当にデータベースに存在するかどうかを確認していないため
修正内容： result.rowCount === 0 のチェックを追加し、存在しないIDへの削除リクエストに 404 を返すように修正
なぜこのバグが起きたか： イレギュラーな操作への対策が抜けていたため
